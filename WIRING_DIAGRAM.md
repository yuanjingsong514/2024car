## 二、循迹功能详解

### 2.1 整体控制架构

```
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────┐
│ 12路灰度  │───→│ sensor_calc  │───→│   P 控制器    │───→│ TB6612   │───→ 左右电机
│ 传感器   │    │ _position()  │    │              │    │ 电机驱动  │
│ UART协议  │    │ 加权求位置   │    │ left =base+diff│    │          │
│          │    │ pos∈[-550,550]│   │ right=base-diff│   │          │
└──────────┘    └──────────────┘    └──────────────┘    └──────────┘
                                             │
                                       每 5ms 循环
                                       (200Hz)
```

### 2.2 传感器数据流

```
传感器(下位机)                        MSPM0G3507
═════════════                        ════════════
                                          
  主动发送 14字节/帧                     UART1 ISR 接收
  ┌─────────────────┐                ┌─────────────────────┐
  │ # 0 1 0 1 ... ! │  ──115200──→  │ sensor_feed_byte()  │
  └─────────────────┘                │ 解析协议:            │
                                     │  '#' → 开始收       │
  S0 S1 S2 ... S11                   │  12字节 → 存 s_buf  │
  左←─────────→右                     │  '!' → g_binary[12] │
                                     │  g_ready = 1        │
                                     └──────┬──────────────┘
                                            │
                                     ┌──────▼──────────────┐
                                     │ sensor_calc_position│
                                     │ 权重:               │
                                     │ S0=-6 S1=-5 ...     │
                                     │ S5=-1 S6=+1 ...     │
                                     │ S11=+6              │
                                     │ pos = sum × 50      │
                                     │ 限幅 ±550           │
                                     └─────────────────────┘
```

### 2.3 P 控制算法

```c
// main.c — 主循环 (每5ms执行一次)
int16_t pos  = sensor_calc_position();   // 黑线位置 [-550, +550]
                                         // 负值=偏左  正值=偏右  0=居中

int16_t diff = (pos * 3) >> 2;           // diff = pos × 0.75 (Kp=0.75)
int16_t base = 200;                      // 基础速度
int16_t left  = base + diff;             // 左轮: 偏右时加速
int16_t right = base - diff;             // 右轮: 偏右时减速
```

**举例说明**：

| 场景 | pos | diff | left | right | 车行为 |
|------|-----|------|------|-------|--------|
| 居中 | 0 | 0 | 200 | 200 | 直行 |
| 偏右 (S8黑) | +150 | +112 | **312** | **88** | 右转 (左快右慢) |
| 偏右 (S10黑) | +250 | +187 | **387** | **13→0** | 急右转 (右轮停) |
| 偏左 (S2黑) | -150 | -112 | **88** | **312** | 左转 (右快左慢) |

> **注意**: 右轮 `13` 落入死区 `(-80, 80)` 会被强制置 0，这就是你观察到的"右轮不转"现象。

### 2.4 电机控制 (TB6612)

**PWM 参数**：
- 频率: 20kHz (32MHz / 1600)
- 占空比: `CC = 1600 - speed × 1600 / 1000`（反相逻辑，CC=0 为 100% 占空比）
- 有效范围: speed ∈ [-1000, 1000]

**方向控制**：

| 动作 | 左电机 AIN1/AIN2 | 右电机 BIN1/BIN2 |
|------|:---:|:---:|
| 前进 | HIGH / LOW | HIGH / LOW |
| 后退 | LOW / HIGH | LOW / HIGH |
| 刹车 | HIGH / HIGH | HIGH / HIGH |
| 滑行 | LOW / LOW | LOW / LOW |

> 左右电机对称安装（物理方向相反），但电气控制方向相同（HIGH/LOW = 前进），因为电机安装时已经翻转。

### 2.5 编码器

```
         A 相 ──┐     ┌──┐     ┌──
                │     │  │     │
                └─────┘  └─────┘
         B 相 ──┐  ┌──┐  ┌──┐
                │  │  │  │  │
                └──┘  └──┘  └──

    A相上升沿 → GPIO中断 → 读B相电平:
      B=HIGH → 正转 → g_enc_count++
      B=LOW  → 反转 → g_enc_count--
```

> 编码器当前已初始化但**尚未接入控制环路**（line_track.h 定义了速度PID结构体但未启用），循迹仅使用纯 P 控制。

### 2.6 遥测输出

每 200ms 通过 UART1 (PB6) 输出一帧：

```
 位置  左PWM  右PWM  传感器状态(12bit)
  ↓      ↓      ↓      ↓
  150  312  088  [001100011000]
  ↑                 ↑↑↑↑↑↑↑↑↑↑↑↑
  pos              S0 S1 ... S11  1=检测到黑线

帧格式: "150 312 88 [001100011000]\r\n"
```

## 三、调试建议

1. **死区 ±80 偏大**：你的 1:28 减速比下，低占空比带不动电机，可考虑改为死区 ±20 + 最低 PWM 保底值（如 100~120）

2. **Kp 可微调**：当前 `Kp=0.75`，如果转弯太猛可以降到 `0.5`（改 `pos * 2 >> 2`），如果转弯不够可以提到 `1.0`（去掉移位直接 `pos`）

3. **base 基础速度**：200 对应 20% 占空比，1:28 减速比下可能偏慢，可适当提高

4. **编码器接入**：后续可将编码器速度反馈接入速度PID，将纯P控制升级为位置环+速度环双闭环（line_track.h 已定义好结构体框架）
